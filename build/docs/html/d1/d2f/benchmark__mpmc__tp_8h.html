Tracepoint for queue operations.

This tracepoint records detailed timing information for each queue operation, enabling post-mortem analysis of latency distribution, contention patterns, and thread behavior.

Parameters
queue_typeName of the queue implementation being tested (e.g., "MutexMPMCQueue", "TwoLockMPMCQueue", "RingBufferMPMC")
operationOperation type ("enqueue" or "dequeue")
thread_idThread identifier executing the operation
latency_nsOperation latency in nanoseconds

@performance The tracepoint overhead is minimal (sub-microsecond) and includes a timestamp capture and string recording.
